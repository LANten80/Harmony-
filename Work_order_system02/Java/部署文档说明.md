## 📋 部署前准备

在开始部署前，请确保你已经准备好：

- [ ] 已购买阿里云ECS服务器（Linux系统）
- [ ] 已获取服务器公网IP地址
- [ ] 已获取服务器登录密码或SSH密钥
- [ ] 已安装SSH客户端（PuTTY、Xshell等）
- [ ] 已安装文件传输工具（WinSCP、FileZilla等）
- [ ] 本地项目可以正常编译运行
- [ ] 已准备好数据库初始化脚本（`database/init.sql`）

## ⚠️ 重要提醒

### 安全建议

1. **修改默认密码**
   - 首次登录后立即修改root密码
   - 修改数据库root密码
   - 使用强密码（至少12位，包含大小写字母、数字、特殊字符）

2. **JWT密钥**
   - 生产环境必须修改JWT密钥
   - 使用至少32位的随机字符串
   - 不要使用默认值或简单字符串

3. **数据库安全**
   - 不要使用root用户连接应用
   - 创建专用数据库用户
   - 使用强密码
   - 生产环境不要开放3306端口到公网

4. **防火墙和安全组**
   - 生产环境建议限制IP访问
   - 不要使用0.0.0.0/0开放所有IP
   - 只开放必要的端口

### 性能建议

1. **服务器配置**
   - 建议至少2核4GB内存
   - 根据实际负载调整配置

2. **JVM参数**
   - 根据服务器内存调整JVM参数
   - 建议：`-Xms512m -Xmx1024m`（2GB内存服务器）

3. **数据库连接池**
   - 根据并发量调整连接池大小
   - 默认配置适合中小型应用

### 维护建议

1. **定期备份**
   - 配置数据库自动备份
   - 定期备份应用文件

2. **监控日志**
   - 定期查看应用日志
   - 监控系统资源使用

3. **更新维护**
   - 定期更新系统补丁
   - 定期更新应用版本

## 📞 获取帮助

如果在部署过程中遇到问题：

1. **查看文档**
   - 查看完整教程中的"常见问题排查"章节
   - 使用检查清单逐项排查

2. **查看日志**
   ```bash
   # 服务日志
   journalctl -u work-order-system -f
   
   # 应用日志
   tail -f /opt/work-order-system/logs/application.log
   ```

3. **检查服务状态**
   ```bash
   systemctl status work-order-system
   systemctl status mysql
   ```

4. **检查网络**
   ```bash
   netstat -tlnp | grep 8080
   firewall-cmd --list-ports
   ```
